# 3. HTTPS (Let's Encrypt)

**Статус:** ⛔ заблокировано — зависит от [пункта 2 (домен)](02-domain-dns.md)

**Задача:** сервис должен открываться только по HTTPS, с настоящим доверенным сертификатом.

## Пошагово
- [ ] Установить Nginx как reverse proxy перед n8n (если ещё не установлен — уже есть на сервере с этапа хостинга сайтов)
- [ ] Создать отдельный server block для поддомена n8n с `proxy_pass` на `http://127.0.0.1:5678`
- [ ] Установить certbot: `sudo apt install certbot python3-certbot-nginx -y`
- [ ] Выпустить сертификат: `sudo certbot --nginx -d n8n.<домен>`
- [ ] Проверить автопродление: `sudo certbot renew --dry-run`
- [ ] Настроить редирект с HTTP на HTTPS (certbot обычно делает это автоматически)
- [ ] Обновить переменные окружения n8n: `N8N_PROTOCOL=https`, `N8N_HOST=n8n.<домен>`, `WEBHOOK_URL=https://n8n.<домен>/`
- [ ] Убрать `N8N_SECURE_COOKIE=false` (больше не нужно — теперь настоящий HTTPS)
- [ ] Порт 5678 больше не должен быть доступен напрямую снаружи — закрыть в `ufw`, доступ только через Nginx (80/443)

## Что нужно узнать
- [ ] Как certbot автоматически продлевает сертификаты (systemd timer, обычно уже настроен из коробки)
- [ ] Разница между reverse proxy на отдельном поддомене и path-based прокси (поддомен проще для n8n — не нужно указывать N8N_PATH)
