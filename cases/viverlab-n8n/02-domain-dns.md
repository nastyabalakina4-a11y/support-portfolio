# 2. Домен

**Статус:** ✅ завершено

**Задача:** настроить доступ к n8n по отдельному домену/поддомену.

## Пошагово
- [x] Выбрать и зарегистрировать домен (`viver-lab.com`, через firstvds — там же, где арендован сервер)
- [x] Создать A-запись поддомена `n8n.viver-lab.com` на IP сервера
- [x] Дождаться распространения DNS
- [x] Проверить: `nslookup n8n.viver-lab.com` возвращает IP сервера

## Что нужно узнать
- [x] Панель firstvds не даёт редактировать DNS-записи для доменов `.com` напрямую — только смену NS-серверов. Решение — перенести управление DNS на **Cloudflare** (бесплатно): добавить сайт в Cloudflare, получить два NS-адреса Cloudflare, прописать их вместо `ns1/ns2.firstvds.ru` в панели firstvds. Сама регистрация домена осталась в firstvds — переехало только управление DNS-записями
- [x] При создании A-записи в Cloudflare важно выключить проксирование (**DNS only**, серое облако) — иначе трафик пойдёт через сеть Cloudflare, что усложнит получение Let's Encrypt сертификата и не нужно для нашего случая

## Что сделано на практике
- Зарегистрирован домен `viver-lab.com`
- В панели firstvds для домена не найдено средств управления DNS-записями (только «Серверы имён» и регистрационные данные) — DNS перенесён на Cloudflare (бесплатный план)
- В Cloudflare добавлена A-запись `n8n` → `157.22.200.6`, режим **DNS only**
- В firstvds на странице «Серверы имён» заменены `ns1.firstvds.ru`/`ns2.firstvds.ru` на `aurora.ns.cloudflare.com`/`wilson.ns.cloudflare.com`
- Cloudflare активировал зону быстро (в пределах часа), проверено через `nslookup n8n.viver-lab.com` → `157.22.200.6`

## Результат
Поддомен `n8n.viver-lab.com` указывает на сервер, DNS полностью рабочий — стало возможным перейти к выпуску SSL-сертификата (см. [этап 3](03-https.md)).

## Проблемы и решения
- Хостинг-провайдер домена (firstvds) не предоставляет DNS zone editor для `.com`-доменов в своей панели — решено переносом DNS на Cloudflare без изменения регистратора самого домена
