# 4. Авторизация и закрытие публичного доступа

**Статус:** ✅ завершено

**Задача:** безопасный вход (логин + надёжный пароль), публичный доступ без авторизации закрыт.

## Пошагово
- [x] Убедиться, что n8n User Management включён (по умолчанию включён — вход невозможен без создания аккаунта)
- [x] Проверить, что пароль владельца (Owner) достаточно надёжный (n8n сам требует минимум 8 символов, минимум одну цифру и одну заглавную букву при создании — задан пароль длиннее и сложнее минимума)
- [x] Убедиться, что без авторизации редактор n8n недоступен
- [ ] ~~(опционально) Двухфакторная аутентификация~~ — не входила в обязательную часть ТЗ, можно добавить отдельно в будущем

## Что нужно узнать
- [x] n8n хранит пароли в виде bcrypt-хэша в своей базе — сам пароль в открытом виде нигде не хранится
- [x] 2FA (TOTP, приложения-аутентификаторы) поддерживается в n8n начиная с Community Edition — можно включить в настройках профиля пользователя при необходимости

## Что сделано на практике
- Проверено практически: открытие `http://<IP>:5678` в приватном окне браузера (без сохранённой сессии) сразу показывает форму **Sign in** (email + пароль) — прямого доступа к редактору без авторизации нет
- Пароль владельца задан с запасом по сложности сверх минимальных требований n8n

## Результат
Публичный доступ без авторизации закрыт — подтверждено практической проверкой в приватном окне браузера, а не просто предположением "раз есть логин, значит защищено".
