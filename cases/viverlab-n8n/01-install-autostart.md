# 1. Установка n8n + автозапуск после перезагрузки

**Статус:** ✅ завершено

**Задача:** актуальная стабильная версия n8n через Docker + Docker Compose, с автоматическим запуском после перезагрузки сервера.

## Пошагово
- [x] Проверить, что установлена актуальная стабильная версия n8n (образ `docker.n8n.io/n8nio/n8n`, тег `latest`)
- [x] Проверить, что сама служба Docker включена в автозагрузку системы (`systemctl is-enabled docker`)
- [x] Убедиться, что контейнер n8n имеет `restart: unless-stopped` в `docker-compose.yml`
- [x] Практически проверить автозапуск — перезагрузить сервер и убедиться, что n8n поднялся сам, без ручного вмешательства

## Что нужно узнать
- [x] Разница между политиками restart в Docker: `no` — никогда не перезапускать; `on-failure` — только при ошибке (падении с ненулевым кодом); `always` — всегда, включая ручную остановку через `docker stop` (перезапустится и после ручной остановки, если не убрать явно); `unless-stopped` — как `always`, но уважает ручную остановку (если контейнер остановили вручную, после перезагрузки Docker он не поднимется сам, пока не запустят заново) — оптимальный вариант для продакшена
- [x] Версию n8n внутри контейнера можно посмотреть командой `docker exec n8n-n8n-1 n8n --version`

## Что сделано на практике
- Проверено: `systemctl is-enabled docker` → `enabled`
- Проверено: `docker inspect n8n-n8n-1 --format '{{.HostConfig.RestartPolicy.Name}}'` → `unless-stopped`
- Сервер перезагружен командой `sudo reboot`
- После перезагрузки и повторного подключения по SSH выполнено `docker compose ps` — контейнер `n8n-n8n-1` уже в статусе `Up`, поднялся автоматически
- В браузере: вход в n8n прошёл без повторного запроса пароля — сессия и все данные сохранились

## Результат
n8n автоматически поднимается после перезагрузки сервера без какого-либо ручного вмешательства — требование ТЗ по автозапуску выполнено и подтверждено практически (не просто "настроено в теории", а проверено реальной перезагрузкой).
