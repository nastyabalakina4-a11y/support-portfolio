# 5. Хранение данных (персистентность)

**Статус:** 🔲 не начато

**Задача:** постоянное хранение всех workflow, credentials, истории запусков и настроек — при обновлении контейнера данные не должны теряться.

## Пошагово
- [ ] Убедиться, что данные n8n хранятся в именованном Docker-томе (`n8n_data`), а не внутри самого контейнера
- [ ] Найти точное имя тома: `docker volume ls`
- [ ] Практически проверить персистентность: создать тестовый workflow → пересоздать контейнер (`docker compose down && docker compose up -d`) → убедиться, что workflow и логин никуда не делись
- [ ] Убедиться, что папка тома доступна для бэкапа (см. [этап 6](06-backups.md))

## Что нужно узнать
- [ ] Что именно хранится в `/home/node/.n8n` внутри контейнера (база SQLite, ключ шифрования credentials, конфиги)
- [ ] Почему нельзя просто пересоздавать контейнер с `docker compose up --force-recreate` без потери данных — потому что volume не удаляется, если явно не указано `-v`/`down -v`
