# 5. Хранение данных (персистентность)

**Статус:** ✅ завершено

**Задача:** постоянное хранение всех workflow, credentials, истории запусков и настроек — при обновлении контейнера данные не должны теряться.

## Пошагово
- [x] Убедиться, что данные n8n хранятся в именованном Docker-томе (`n8n_data`), а не внутри самого контейнера
- [x] Найти точное имя тома: `docker volume ls`
- [x] Практически проверить персистентность: пересоздать контейнер и убедиться, что аккаунт/данные никуда не делись (проверено пересозданием через перезагрузку сервера в [пункте 1](01-install-autostart.md) — контейнер полностью пересоздаётся при старте Docker, данные сохранились)
- [x] Убедиться, что папка тома доступна для бэкапа

## Что нужно узнать
- [x] В `/home/node/.n8n` внутри контейнера хранится: база SQLite (все workflow, credentials в зашифрованном виде, история выполнений), файл `config` с ключом шифрования credentials
- [x] Volume не удаляется при обычном пересоздании контейнера (`docker compose down` без флага `-v`, `docker compose up -d`, перезагрузка сервера) — удаляется только явной командой с `-v` или `docker volume rm`, поэтому случайно данные не потеряются

## Что сделано на практике
- Найдено точное имя тома: `docker volume ls` → `n8n_n8n_data` (формат `<имя_проекта>_<имя_тома_из_compose>`)
- Персистентность подтверждена практически: после перезагрузки сервера (пункт 1) контейнер пересоздался, но вход в аккаунт прошёл без повторной настройки — все данные сохранились в volume

## Результат
Все данные n8n (workflow, credentials, история запусков, настройки) хранятся в постоянном Docker-томе `n8n_n8n_data`, не зависят от жизненного цикла самого контейнера — пересоздание/обновление контейнера данные не затрагивает.
