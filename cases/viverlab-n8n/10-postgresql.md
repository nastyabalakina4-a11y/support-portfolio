# 10. Миграция на PostgreSQL

**Статус:** 🔲 не начато

**Задача:** перевести n8n с встроенной SQLite на полноценную PostgreSQL — требование обновлённого ТЗ (важно перед серьёзной нагрузкой от автоматизаций публикаций и аналитики).

## Пошагово
- [ ] Добавить сервис `postgres` в `docker-compose.yml` (образ `postgres:16-alpine`, отдельный volume для данных БД)
- [ ] Настроить переменные окружения n8n для подключения к PostgreSQL (`DB_TYPE=postgresdb`, `DB_POSTGRESDB_HOST`, `DB_POSTGRESDB_DATABASE`, `DB_POSTGRESDB_USER`, `DB_POSTGRESDB_PASSWORD`)
- [ ] Продумать перенос существующих данных из SQLite в PostgreSQL (n8n поддерживает миграцию через штатный механизм — нужно свериться с официальной документацией на момент выполнения) либо начать с чистой БД, если переносить нечего критичного
- [ ] Обновить скрипт бэкапа — теперь нужно бэкапить и volume PostgreSQL (`pg_dump` предпочтительнее прямого архивирования файлов БД)
- [ ] Проверить персистентность и процедуру обновления заново (docker compose down/up) уже с PostgreSQL

## Что нужно узнать
- [ ] Официальная процедура миграции данных n8n SQLite → PostgreSQL (или решение начать с чистой БД)
- [ ] Разница в бэкапе: `pg_dump`/`pg_dumpall` (логический бэкап, независимый от версии) vs бэкап самого volume с файлами БД (проще, но менее переносимо)
