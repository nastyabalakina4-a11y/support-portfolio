# 7. Безопасное обновление

**Статус:** 🔲 не начато

**Задача:** возможность безопасно обновлять n8n без потери данных.

## Пошагово
- [ ] Задокументировать пошаговую процедуру обновления: бэкап → `docker compose pull` → `docker compose up -d` → проверка, что всё работает
- [ ] Проверить процедуру практически: сделать бэкап, обновить образ, убедиться, что workflow/credentials/логин сохранились
- [ ] Определить план отката (rollback) — что делать, если новая версия не заработала (вернуть предыдущий тег образа + восстановить бэкап при необходимости)

## Что нужно узнать
- [ ] Как посмотреть changelog/breaking changes перед обновлением на новую мажорную версию n8n
- [ ] Разница между тегом `latest` (всегда последняя версия) и фиксацией на конкретной версии (`n8nio/n8n:1.6x.x`) — фиксированная версия безопаснее для продакшена, обновление тогда осознанное действие, а не случайное при пересоздании контейнера
