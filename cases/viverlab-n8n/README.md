# Кейс: Продакшен-настройка n8n по ТЗ (Viver Lab)

> 🚧 Кейс в процессе. Базовая инфраструктура (пункты 1-7 из первой версии ТЗ) полностью готова. ТЗ было расширено — сейчас в работе интеграции с ИИ и подготовка к автопостингу.

## Контекст

Личный/командный проект (Viver Lab) — после первого пробного запуска n8n на сервере ([см. соответствующий этап основного кейса](../own-server-setup/08-n8n-automation.md)) появилось формальное техническое задание: довести установку до продакшен-уровня. По ходу работы ТЗ было расширено до полноценной платформы автоматизации с PostgreSQL, ИИ-интеграциями и автопостингом.

## Техническое задание (актуальная версия)

**Цель:** развернуть масштабируемую платформу автоматизации на базе n8n для Viver Lab с интеграцией ИИ, Notion, Telegram, Google-сервисов и других систем.

1. **Инфраструктура** — n8n через Docker + Docker Compose, PostgreSQL вместо SQLite, автозапуск после перезагрузки, отдельный домен и HTTPS (Let's Encrypt), защита доступа авторизацией
2. **Хранение данных и резервное копирование** — workflow, credentials, история запусков, БД PostgreSQL; автоматические бэкапы; безопасное обновление без потери данных
3. **Автоматизация публикаций** — по расписанию и вручную, в несколько площадок одновременно, текст/изображения/видео/документы/ссылки, повторная публикация
4. **Источник данных** — единый источник контента (Notion или Google Sheets): ID, текст, заголовок, язык, автор, площадки, дата, статус, вложения
5. **Логирование и отчёты** — дата, площадка, статус, ссылка на пост, ID публикации, ошибки; экспорт в Google Sheets/Excel/CSV
6. **Аналитика с ИИ** — OpenAI (ChatGPT) или Anthropic (Claude): анализ эффективности публикаций, рекомендации, отчёты, лучшее время публикации, сохранение результатов в Notion/Google Docs/Sheets
7. **Использование ИИ в workflow** — анализ результатов, идеи новых материалов, генерация контента, автоотчёты, решения внутри сценариев на основе статистики
8. **Архитектура автоматизаций** — генерация контента, перевод, автопостинг, база знаний, CRM и продажи, онбординг сотрудников, внутренние процессы
9. **Соцсети и интеллектуальный автопостинг** — публикация в несколько площадок сразу, выбор площадок, автоперевод, адаптация текста под площадку через ИИ, отдельные заголовки/описания/CTA, оптимизация длины/хештегов, публикация с учётом часовых поясов, автовыбор времени по статистике, сбор ссылок и статистики, подключение новых платформ без переработки системы
10. **Требования** — стабильная работа 24/7, масштабируемость, простое администрирование, безопасность данных, возможность подключения новых сервисов
11. **После завершения работ предоставить** — URL, логин и пароль, инструкцию по обновлению, инструкцию по восстановлению, описание структуры контейнеров и расположения бэкапов

**Планируемые интеграции:** Notion, OpenAI API, Anthropic API, Telegram Bot API, Google Drive/Docs/Sheets/Gmail/Calendar, Tilda, Figma, GitHub.

## Моя роль

Самостоятельно довожу практический запуск n8n до продакшен-уровня по формальному ТЗ, включая обход инфраструктурных препятствий (например, блокировку OpenAI/Anthropic API для российских IP).

## Ключевые навыки и инструменты

* Docker / Docker Compose (продакшен-эксплуатация)
* Персистентные данные и volume-бэкапы в Docker
* Nginx reverse proxy, Let's Encrypt / certbot
* DNS-администрирование (Cloudflare, перенос NS между провайдерами)
* Обход геоблокировок через собственный прокси-сервер (Nginx reverse proxy к API за рубежом)
* cron, автоматизация резервного копирования
* Планирование инфраструктуры по формальному (и меняющемуся) ТЗ
* Разграничение доступа для сторонних людей (ограниченный SSH-пользователь без sudo для деплоя коллегой статичного сайта)
