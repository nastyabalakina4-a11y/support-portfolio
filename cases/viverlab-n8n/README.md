# Кейс: Продакшен-настройка n8n по ТЗ (Viver Lab)

> 🚧 Кейс в процессе. Часть работы выполнена, часть заблокирована до покупки домена (отмечено в плане).

## Контекст

Личный/командный проект (Viver Lab) — после первого пробного запуска n8n на сервере ([см. соответствующий этап основного кейса](../own-server-setup/08-n8n-automation.md)) появилось формальное техническое задание: довести установку до продакшен-уровня — с доменом, HTTPS, персистентным хранением данных, резервным копированием и безопасной процедурой обновлений. Дальше на этой базе планируется подключать реальные интеграции (Notion, ChatGPT, Claude, Telegram, Google Workspace и др.) и автоматизации для команды.

## Техническое задание

**Что необходимо настроить:**
1. Установка n8n — актуальная стабильная версия, Docker + Docker Compose, автозапуск после перезагрузки сервера
2. Домен — доступ по отдельному домену/поддомену (например, `n8n.viverlab.ru`)
3. HTTPS — SSL-сертификат (Let's Encrypt), сервис должен открываться только по HTTPS
4. Авторизация — логин + надёжный пароль, закрыт публичный доступ без авторизации
5. Хранение данных — постоянное хранение workflow, credentials, истории запусков, настроек; при обновлении контейнера данные не теряются
6. Резервное копирование — регулярный backup базы данных, конфигурации, пользовательских данных, желательно автоматический
7. Обновления — безопасное обновление n8n без потери данных

**После установки нужно предоставить:** URL для входа, логин, пароль, где лежат резервные копии, инструкцию по обновлению.

**Планируемые интеграции:** Notion, ChatGPT (OpenAI API), Claude (Anthropic API), Telegram Bot API, Google Drive/Docs/Sheets/Gmail/Calendar, Tilda, Figma, GitHub.

**Планируемые автоматизации:** автосоздание контента, перевод на несколько языков, автопостинг в соцсети, обработка обучающих материалов, генерация конспектов/чек-листов/FAQ, база знаний в Notion, автоматизация внутренних процессов команды, онбординг, продажи/CRM, клиентские проекты Viver Lab.

**Требования:** стабильная работа 24/7, возможность масштабирования, простота дальнейшего администрирования, возможность подключать новые сервисы без переустановки системы.
