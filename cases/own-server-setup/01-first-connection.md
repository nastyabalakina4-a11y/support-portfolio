# Этап 1. Первое подключение

**Статус:** ✅ завершено

**Задача:** подключиться к серверу по SSH.

## Что делать
- [x] Установить на свой компьютер SSH-клиент (на Windows — Windows Terminal/PowerShell уже умеет `ssh`, на телефоне — Termux или JuiceSSH)
- [x] Подключиться: `ssh root@ваш_ip`
- [x] Ввести пароль из письма хостера

## Что нужно узнать
- [ ] Разница между подключением по паролю и по SSH-ключу
- [ ] Что такое `known_hosts` и почему при первом подключении SSH спрашивает про fingerprint

## Что сделано на практике
- Выполнено первое подключение к серверу по SSH под `root` с паролем из личного кабинета хостинга
- Обновлена система: `apt update && apt upgrade -y`

## Результат
Подключение к серверу установлено, система обновлена до актуального состояния.

## Проблемы и решения
- Во время `apt upgrade` пересобрался и перезапустился SSH-сервис, сессия оборвалась — переподключились без потери данных
