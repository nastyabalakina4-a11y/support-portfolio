# Этап 8. Автоматизация процессов (n8n)

**Статус:** ✅ завершено

**Задача:** развернуть n8n для автоматизации рабочих процессов и интеграций между сервисами, с доступом для второго человека (подруги).

## Пошагово
- [x] Установить Docker и Docker Compose на сервере (VPS, не ПК — n8n должен быть доступен независимо от того, включён ли домашний компьютер)
- [x] Подготовить `docker-compose.yml` для n8n с постоянным томом для данных
- [x] Настроить переменные окружения (хост, порт, протокол, часовой пояс)
- [x] Запустить контейнер n8n
- [ ] ~~Настроить Nginx как реверс-прокси + HTTPS (Let's Encrypt)~~ — отложено, для Let's Encrypt нужен домен, а домена пока нет
- [x] Открыть порт n8n в `ufw`, доступ по HTTP с защитой логином/паролем самого n8n
- [x] Проверить работу через браузер, создать аккаунт владельца
- [x] Настроить multi-user — пригласить второго пользователя (подругу) с ролью Member

## Что нужно узнать
- [x] Нужен ли отдельный домен/поддомен для n8n — да, для нормального HTTPS через Let's Encrypt; пока работаем по IP по HTTP
- [ ] Как настроить бэкап данных n8n (том с базой SQLite)
- [x] Основы multi-user в n8n: роль Owner создаётся первой при установке, дополнительные пользователи с ролью Member добавляются бесплатно (Community Edition), доп. Admin-роли — только в платной версии

## Что сделано на практике
- Установлен Docker Engine + Docker Compose plugin на VPS (официальный репозиторий Docker для Ubuntu)
- Создан `~/n8n/docker-compose.yml` с образом `docker.n8n.io/n8nio/n8n`, портом 5678, постоянным томом `n8n_data` для данных и настройками (`N8N_HOST`, `N8N_PORT`, `N8N_PROTOCOL=http`, `WEBHOOK_URL`, `GENERIC_TIMEZONE=Europe/Moscow`, `N8N_SECURE_COOKIE=false` — без этого логин не работает по обычному HTTP, так как cookie сессии по умолчанию требует HTTPS)
- Порт 5678 открыт в `ufw`
- Контейнер запущен (`docker compose up -d`), создан аккаунт владельца (Owner)
- В `Settings → Users` создано приглашение для второго пользователя с ролью **Member** (бесплатно в Community Edition, доп. Admin-аккаунты — платная функция) — так как SMTP на сервере не настроен, n8n не отправляет письмо само, а генерирует ссылку-приглашение, которую нужно переслать вручную

## Результат
n8n развёрнут на VPS и работает независимо от домашнего ПК. Двое пользователей (я и подруга) могут заходить и строить автоматизации по адресу `http://157.22.200.6:5678`, каждый под своим аккаунтом.

## Проблемы и решения
- Сразу после запуска контейнера сайт отдавал `{"code":503,"message":"Database is not ready!"}` — n8n ещё выполнял миграции внутренней базы (SQLite) при первом старте. Решение — просто подождать (около минуты) и обновить страницу, в логах (`docker compose logs`) в этот момент видно `Editor is now accessible via: ...`
- Без домена нет HTTPS от Let's Encrypt — оставили доступ по обычному HTTP с логином/паролем n8n; при появлении домена стоит донастроить Nginx как реверс-прокси с HTTPS
