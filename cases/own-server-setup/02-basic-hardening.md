# Этап 2. Базовая защита сервера

**Статус:** 🔲 не начато

**Задача:** закрыть очевидные дыры до того, как сервер увидят боты-сканеры (а они найдут его в течение часов после включения).

## Пошагово
- [ ] Обновить систему: `apt update && apt upgrade -y`
- [ ] Создать обычного пользователя с правами sudo (не работать постоянно под root):
  `adduser ваше_имя` → `usermod -aG sudo ваше_имя`
- [ ] Настроить вход по SSH-ключу вместо пароля:
  - Сгенерировать ключ на своём компьютере (`ssh-keygen`)
  - Скопировать публичный ключ на сервер (`ssh-copy-id`)
- [ ] Отключить вход по паролю и вход под root в файле `/etc/ssh/sshd_config`:
  - `PasswordAuthentication no`
  - `PermitRootLogin no`
- [ ] Сменить порт SSH с 22 на нестандартный (снижает шум от автоматических атак)
- [ ] Установить `fail2ban` — блокирует IP после нескольких неудачных попыток входа
- [ ] Настроить firewall (`ufw`):
  - Разрешить только нужные порты (новый SSH-порт, позже — VPN, SIP)
  - Всё остальное — закрыть

## Что нужно узнать
- [ ] Что такое SSH-ключи (публичный/приватный) и почему это безопаснее пароля
- [ ] Как не потерять доступ к серверу, если ошибётесь в конфиге SSH (совет: держите сессию открытой, пока не проверите новую в отдельном окне)
- [ ] Базовые команды `ufw`: allow, deny, status
