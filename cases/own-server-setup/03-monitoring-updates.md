# Этап 3. Автообновления и базовый мониторинг

**Статус:** 🔲 не начато

**Задача:** чтобы сервер не "протух" без вашего участия.

## Пошагово
- [ ] Настроить автоматические обновления безопасности: `unattended-upgrades`
- [ ] Настроить ротацию логов (`logrotate`) — чтобы логи не забили диск
- [ ] Поставить `htop` и `ncdu` для просмотра нагрузки и занятого места на диске

### Мониторинг через Zabbix
- [ ] Подключить официальный репозиторий Zabbix и установить пакеты: `zabbix-server`, `zabbix-frontend-php`, `zabbix-agent` (или `zabbix-agent2`)
- [ ] Установить и настроить СУБД для Zabbix (MySQL/PostgreSQL)
- [ ] Импортировать начальную схему базы данных
- [ ] Настроить `/etc/zabbix/zabbix_server.conf` (подключение к БД)
- [ ] Настроить веб-фронтенд Zabbix (через Nginx/Apache + PHP)
- [ ] Открыть нужные порты в `ufw` (10051 — сервер, 10050 — агент, порт веб-интерфейса)
- [ ] Запустить `zabbix-server`, `zabbix-agent` и зависимые службы, добавить в автозагрузку
- [ ] Зайти в веб-интерфейс, сменить пароль администратора по умолчанию
- [ ] Добавить сам сервер как узел сети (host) для мониторинга
- [ ] Настроить триггеры/оповещения (например, email или Telegram) на критичные события (диск, память, недоступность SSH и т.п.)

## Что нужно узнать
- [ ] Разница между обновлениями безопасности и полным обновлением системы (второе иногда лучше делать вручную)
- [ ] Как посмотреть, сколько места на диске и что его ест (`df -h`, `du -sh`)
- [ ] Хватит ли ресурсов (2 ГБ RAM) для полного стека Zabbix server + frontend + БД на этом же VPS, или лучше облегчённый вариант
- [ ] Разница между `zabbix-agent` и `zabbix-agent2`
- [ ] Как безопасно открыть веб-интерфейс Zabbix наружу — ограничить по IP, доступ только через уже настроенный WireGuard-туннель, либо реверс-прокси с базовой авторизацией

## Что сделано на практике
Внедрение ещё не начато, но изучены теоретически (без установки/настройки):
- Критерии, по которым можно оценить, потянет ли сервер конкретный сайт: тип сайта (статика/база данных/медиа), количество одновременных посетителей
- Инструменты мониторинга нагрузки: `htop`, `uptime` (load average), `free -h`
