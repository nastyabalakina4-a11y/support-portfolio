# Этап 3. Автообновления и базовый мониторинг

**Статус:** 🔲 не начато

**Задача:** чтобы сервер не "протух" без вашего участия.

## Пошагово
- [ ] Настроить автоматические обновления безопасности: `unattended-upgrades`
- [ ] Настроить ротацию логов (`logrotate`) — чтобы логи не забили диск
- [ ] Поставить `htop` и `ncdu` для просмотра нагрузки и занятого места на диске

### Мониторинг через Zabbix

**Цель:** учебная практика — разобраться, как устроен Zabbix и как он работает, чтобы указать в резюме базовое знание инструмента. Не production-мониторинг 24/7.

**Архитектура:** Zabbix Server (+ БД + веб-фронтенд) ставится на домашний ПК, на VPS — только лёгкий Zabbix Agent. Связь — passive-проверки: Server сам подключается к Agent на сервере (не наоборот), это не требует проброса портов на домашнем роутере.

- [ ] На домашнем ПК: установить Zabbix Server, СУБД (MySQL/PostgreSQL) и веб-фронтенд
- [ ] Импортировать начальную схему базы данных, настроить `zabbix_server.conf`
- [ ] Зайти в веб-интерфейс, сменить пароль администратора по умолчанию
- [ ] На VPS: подключить репозиторий Zabbix, установить `zabbix-agent` (или `zabbix-agent2`)
- [ ] На VPS: открыть порт 10050/tcp в `ufw`, ограничив доступ по IP домашнего ПК (`ufw allow from ВАШ_IP to any port 10050`)
- [ ] Добавить VPS как узел сети (host) в веб-интерфейсе Zabbix
- [ ] Посмотреть собираемые метрики (CPU, память, диск, сеть) на графиках
- [ ] Для практики настроить один-два триггера/оповещения (например, на заполнение диска) — просто чтобы понять, как это работает

## Что нужно узнать
- [ ] Разница между обновлениями безопасности и полным обновлением системы (второе иногда лучше делать вручную)
- [ ] Как посмотреть, сколько места на диске и что его ест (`df -h`, `du -sh`)
- [ ] Разница между `zabbix-agent` и `zabbix-agent2`
- [ ] Базовые понятия Zabbix: host, item, trigger, action — чтобы уверенно объяснить их на собеседовании

## Что сделано на практике
Внедрение ещё не начато, но изучены теоретически (без установки/настройки):
- Критерии, по которым можно оценить, потянет ли сервер конкретный сайт: тип сайта (статика/база данных/медиа), количество одновременных посетителей
- Инструменты мониторинга нагрузки: `htop`, `uptime` (load average), `free -h`
