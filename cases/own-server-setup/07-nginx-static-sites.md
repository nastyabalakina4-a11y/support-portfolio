# Этап 7. Хостинг статичных сайтов (Nginx)

**Статус:** ✅ завершено

**Задача:** использовать сервер как площадку для размещения статичных HTML-сайтов.

## Пошагово
- [x] Установить Nginx: `apt install nginx`
- [x] Разместить первый статичный HTML-сайт
- [x] Настроить отдельный virtual host (server block) вместо редактирования дефолтной страницы
- [x] Порты 80/443 уже открыты в firewall (см. [этап 2](02-basic-hardening.md))

## Что нужно узнать
- [ ] Критерии, по которым можно оценить, потянет ли сервер конкретный сайт: тип сайта (статика/база данных/медиа), количество одновременных посетителей
- [x] Инструменты мониторинга нагрузки на сервер: `htop`, `uptime` (load average), `free -h` — освоено на этапе 3 через Zabbix

## Что сделано на практике
- Установлен Nginx (`apt install nginx`), проверена работа стандартной приветственной страницы через IP сервера
- Создан отдельный каталог сайта `/var/www/testsite/html` вместо использования `/var/www/html` по умолчанию
- Настроен отдельный конфиг сайта (`/etc/nginx/sites-available/testsite`) с собственным `root`, `index` и `location`
- Старый дефолтный сайт отключён (`sites-enabled/default` удалён), новый подключён через симлинк и назначен `default_server` — так как домена пока нет, сайт открывается напрямую по IP
- Конфигурация проверена (`nginx -t`) и применена без остановки сервиса (`systemctl reload nginx`)
- Тестовая страница заменена на полноценный лендинг проекта («Настройка сервера для работы») — карточки по каждому завершённому этапу с SVG-иконками, тёмная тема, адаптивная вёрстка (grid), без внешних зависимостей (CSS и JS полностью в одном файле)
- Добавлен JS-lightbox: часть чипов технологий (SSH, ufw/fail2ban, WireGuard, Zabbix) стали кликабельными кнопками, открывающими по клику схему-инфографику соответствующего этапа поверх страницы (оверлей, закрытие по клику вне картинки, кнопке × или Esc)

## Результат
Сайт открывается по IP сервера через собственный virtual host. Домена пока нет — при появлении достаточно поменять `server_name _;` на реальное имя и (при необходимости) добавить HTTPS через Let's Encrypt/certbot.

Исходный код лендинга: [site/index.html](site/index.html)

## Проблемы и решения
- При первой правке дефолтной страницы вручную обнаружилась «кракозябра» вместо кириллицы — браузер не знал кодировку страницы. Решение — добавить `<meta charset="UTF-8">` в `<head>`
- При очистке файла в `nano` через `Ctrl+K` часть старого текста осталась, и новый текст задвоился — потребовалось почистить файл полностью (несколько `Ctrl+K` подряд, либо удалить и создать файл заново)
