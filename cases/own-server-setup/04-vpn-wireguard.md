# Этап 4. Настройка VPN (WireGuard)

**Статус:** ✅ завершено

**Задача:** получить доступ к российским сервисам из-за границы.

## Пошагово
- [x] Установить WireGuard: `apt install wireguard`
- [x] Сгенерировать ключи сервера и клиента (`wg genkey`, `wg pubkey`)
- [x] Настроить конфиг сервера (`/etc/wireguard/wg0.conf`) — IP-адрес туннеля, порт, ключи
- [x] Включить IP-forwarding в `/etc/sysctl.conf`
- [x] Настроить NAT-правила (iptables/nftables), чтобы трафик клиента шёл в интернет через сервер
- [x] Открыть нужный порт в `ufw`
- [x] Создать конфиг для клиента (телефон/ноутбук) — можно сгенерировать QR-код для удобного подключения с телефона
- [x] Проверить: подключиться с зарубежного устройства и зайти на сайт, который обычно недоступен

## Что нужно узнать
- [ ] Как работает split-tunneling (весь трафик через VPN vs только выборочные домены/IP)
- [x] Как проверить, что ваш IP при подключении действительно российский (сайты типа 2ip.ru)
- [ ] Что делать, если провайдер за границей блокирует протокол WireGuard (запасной вариант — VLESS+Reality или Shadowsocks)

## Что сделано на практике
- Установлен WireGuard, сгенерированы ключевые пары для сервера и клиента
- Настроен конфиг сервера: собственная подсеть `10.8.0.0/24` для VPN-туннеля
- Включён `net.ipv4.ip_forward` для маршрутизации трафика
- Настроен NAT (masquerade) через `iptables`, чтобы трафик клиента "маскировался" под IP сервера
- Открыт порт `51820/udp` в firewall
- Клиентский конфиг импортирован в официальное приложение WireGuard для Windows

## Результат
Туннель проверен практически: после активации VPN проверка через 2ip.ru показала IP и геолокацию сервера (Россия) вместо реального зарубежного IP.

## Проблемы и решения
- Подсеть туннеля нужно подбирать так, чтобы не пересекаться с реальной сетью сервера — на старте по ошибке использовали диапазон, совпадающий с шлюзом сервера, вовремя заметили и исправили
- Один из приватных ключей случайно попал в общий терминальный вывод при генерации — пересоздали ключевую пару заново и удалили старые файлы, чтобы не использовать потенциально скомпрометированный ключ
